## Brand Overview

### Mission Statement
MenuMaker empowers food entrepreneurs to launch and grow their businesses online, making it simple for anyone to turn their culinary passion into profit.

### Brand Personality
- **Approachable**: Friendly and welcoming, never intimidating
- **Empowering**: Gives sellers confidence and control
- **Efficient**: Streamlined, no-nonsense tools
- **Trustworthy**: Reliable partner for business growth
- **Innovative**: Forward-thinking without being overly technical

### Target Audience
- **Primary**: Home-based food sellers, cloud kitchens, small restaurants (India-focused)
- **Secondary**: Food bloggers, influencers, aspiring entrepreneurs
- **Age**: 25-45 years old
- **Tech Savvy**: Moderate (comfortable with smartphones, not necessarily tech experts)

## Color Palette

### Primary Colors

**MenuMaker Orange** (#FF6B35)
- **Usage**: Primary CTAs, links, active states, brand accents
- **Emotional Association**: Energy, appetite, warmth
- **Accessibility**: Use `primary-600` (#E65620) for text on white backgrounds (better contrast)

**Trust Blue** (#004E89)
- **Usage**: Secondary buttons, navigation, informational elements
- **Emotional Association**: Trust, professionalism, stability
- **Accessibility**: Sufficient contrast for text on white

---

### Functional Colors

**Success Green** (#28A745)
- Order confirmations, success messages, positive metrics

**Warning Yellow** (#FFC107)
- Warnings, pending states, alerts (non-critical)

**Error Red** (#DC3545)
- Errors, destructive actions, critical alerts

---

### Neutral Palette

**Usage**: Text, borders, backgrounds, shadows

| Color | Hex | Usage |
|-------|-----|-------|
| White | #FFFFFF | Page backgrounds, card surfaces |
| Neutral 50 | #F8F9FA | Alternate backgrounds |
| Neutral 200 | #E9ECEF | Borders, dividers |
| Neutral 600 | #6C757D | Secondary text |
| Neutral 900 | #212529 | Primary text, headings |
| Black | #000000 | Reserved for special emphasis |

---

### Color Combinations

**Approved Pairings**:
- Primary Orange + White (high impact CTAs)
- Trust Blue + White (informational sections)
- Neutral 900 + Neutral 50 (text on background)

**Avoid**:
- Orange + Blue (too much brand color conflict)
- Orange + Red (confusing for error states)
- Light text on light backgrounds (accessibility)

## Typography

### Font Families

**Primary Font: Inter**
- **Usage**: UI text, body copy, headings
- **Weights**: Regular (400), Medium (500), Semibold (600), Bold (700)
- **Source**: Google Fonts
- **Fallback**: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif

**Monospace Font: JetBrains Mono**
- **Usage**: Code snippets, API keys, technical data
- **Weights**: Regular (400), Medium (500)
- **Source**: Google Fonts

---

### Typography Scale

| Name | Size | Line Height | Usage |
|------|------|-------------|-------|
| xs | 12px | 1.5 | Helper text, labels |
| sm | 14px | 1.5 | Body text (small), captions |
| base | 16px | 1.5 | Body text (default) |
| lg | 18px | 1.5 | Large body text, intros |
| xl | 20px | 1.5 | Small headings |
| 2xl | 24px | 1.2 | Section headings (H3) |
| 3xl | 30px | 1.2 | Page headings (H2) |
| 4xl | 36px | 1.2 | Hero headings (H1) |
| 5xl | 48px | 1.2 | Landing page heroes |

---

### Font Weight Usage

- **Regular (400)**: Body text, paragraphs
- **Medium (500)**: Emphasized text, labels
- **Semibold (600)**: Subheadings, button text
- **Bold (700)**: Headings, critical information

---

### Typography Rules

1. **Hierarchy**: Use size + weight to establish clear content hierarchy
2. **Line Length**: Max 75 characters per line for readability
3. **Line Height**: 1.5 for body text, 1.2 for headings
4. **Alignment**: Left-aligned for body text, centered for headings (optional)
5. **Contrast**: Minimum 4.5:1 ratio for text (WCAG AA)

## Voice & Tone

### Brand Voice

**Principles**:
1. **Friendly**: Use conversational language, avoid corporate jargon
2. **Helpful**: Guide users, don't just instruct
3. **Confident**: Direct and clear, not hesitant
4. **Inclusive**: Welcoming to all skill levels
5. **Non-technical**: Explain concepts simply

---

### Tone by Context

**Onboarding / Tutorials**
- **Tone**: Encouraging, patient
- **Example**: "Let's create your first menu! This will only take a minute."
- **Avoid**: "Complete the mandatory menu setup process."

**Success Messages**
- **Tone**: Celebratory, rewarding
- **Example**: "Amazing! Your menu is live and ready for orders! üéâ"
- **Avoid**: "Menu published successfully."

**Error Messages**
- **Tone**: Apologetic, solution-focused
- **Example**: "Oops! We couldn't save your changes. Please check your internet connection and try again."
- **Avoid**: "Error 500: Internal server error."

**Educational Content**
- **Tone**: Informative, not condescending
- **Example**: "Tip: Adding high-quality photos increases orders by 30%."
- **Avoid**: "You must add photos to improve conversion rates."

**Marketing Copy**
- **Tone**: Inspiring, empowering
- **Example**: "Turn your kitchen into a business. Start selling in minutes, not months."
- **Avoid**: "Leverage our platform to monetize your culinary offerings."

---

### Writing Guidelines

**Do**:
- Use active voice ("Create your menu" vs. "Your menu can be created")
- Address users directly ("You can now..." vs. "Users can now...")
- Use contractions ("you're", "we'll") for friendliness
- Break up long text with bullet points
- Use emojis sparingly for warmth (success messages, celebrations)

**Don't**:
- Use jargon (API, backend, database) unless in developer docs
- Write in ALL CAPS (except acronyms)
- Use exclamation marks excessively (max 1 per message)
- Assume technical knowledge
- Be overly formal or stiff

## Imagery

### Photography Style

**Subject Matter**:
- **Food**: Fresh, appetizing, well-lit dishes
- **People**: Real sellers and customers (diverse, authentic)
- **Environment**: Kitchens, restaurants, delivery scenarios

**Visual Style**:
- **Lighting**: Natural light preferred, soft shadows
- **Composition**: Rule of thirds, close-ups for food
- **Colors**: Warm tones (orange, yellow) to stimulate appetite
- **Focus**: Sharp focus on subject, soft bokeh background

**Examples**:
- ‚úÖ Vibrant biryani bowl, steam rising, natural window light
- ‚úÖ Smiling chef presenting freshly baked bread
- ‚ùå Dark, poorly lit food photos
- ‚ùå Stock photos with watermarks

---

### Image Guidelines

**Resolution**:
- Web: Minimum 1200px width
- Social media: Platform-specific (Instagram 1080x1080, Facebook 1200x630)
- Print: 300 DPI minimum

**Formats**:
- Photos: JPEG (web), PNG (transparency needed)
- Logos: SVG (preferred), PNG (fallback)
- Icons: SVG

**Optimization**:
- Compress images (50-80% quality for web)
- Use lazy loading for below-the-fold images
- Provide `alt` text for accessibility

---

### Illustration Style

**When to Use Illustrations**:
- Empty states ("No orders yet")
- Onboarding flows
- Error pages (404, 500)
- Feature explanations

**Style**:
- **Hand-drawn aesthetic**: Warm, approachable
- **Color palette**: Primary orange, neutral grays, accent colors
- **Line weight**: Medium (2-3px)
- **Complexity**: Simple, not overly detailed

## Marketing Materials

### Social Media

**Profile Images**:
- Use icon-only logo
- Square format (1:1 ratio)
- High resolution (512x512px minimum)

**Cover Photos**:
- Hero image + tagline overlay
- Facebook: 820x312px
- Twitter: 1500x500px
- LinkedIn: 1128x191px

**Post Templates**:
- Feature announcements: Orange gradient background + screenshot
- Customer success stories: Photo + testimonial quote
- Tips & tricks: Illustration + text overlay

---

### Email Templates

**Structure**:
- Header: Logo + navigation
- Body: Max 600px width, left-aligned text
- Footer: Unsubscribe link, social links, legal info

**Typography**:
- Subject line: 50 characters max
- Preheader: 90 characters max
- Body: 16px Inter, line-height 1.5

---

### Print Materials

**Business Cards**:
- Front: Logo + tagline
- Back: Contact info + QR code
- Size: 3.5" x 2" (standard)

**Flyers**:
- A5 size (148mm x 210mm)
- High-quality food photography
- Clear call-to-action
- QR code for app download

## Do's and Don'ts

### Visual Design

‚úÖ **Do**:
- Maintain consistent spacing (8px grid)
- Use high-contrast color combinations
- Test designs in dark mode
- Optimize images for web performance
- Follow accessibility guidelines (WCAG AA)

‚ùå **Don't**:
- Use more than 3 font sizes in one design
- Place text directly on photos (use overlay)
- Use low-resolution images
- Ignore mobile responsiveness
- Sacrifice accessibility for aesthetics

---

### Content

‚úÖ **Do**:
- Write in active voice
- Use inclusive language
- Proofread before publishing
- Localize content for target markets
- Test copy with real users

‚ùå **Don't**:
- Use discriminatory language
- Make promises you can't keep
- Copy competitors' content
- Use clickbait headlines
- Ignore user feedback

---

## Brand Review Process

Before publishing any branded material:

1. **Self-Review**: Check against these guidelines
2. **Peer Review**: Get feedback from design team
3. **Accessibility Check**: Test with screen readers, color contrast tools
4. **Approval**: Final sign-off from brand manager
5. **Archive**: Save approved assets to brand repository
